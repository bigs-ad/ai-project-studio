# 工作项规格：标题

## 目标

## 非目标

- 明确本次不解决什么。

## 验收条件

- 可观察、可判断的条件。

## 影响范围

- 涉及的模块、内容或资产。

## 验证方式

- 测试命令、构建路径、截图、录屏或试玩观察。

## 当前阶段盲区

- 仅列与本工作项有关、可能影响判断的未知和专业缺口。
